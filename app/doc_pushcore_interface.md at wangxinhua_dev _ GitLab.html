pushcore 接口文档.

0. 协议.

pushcore 对客户端提供 websocket 协议的接口, 数据格式统一为 json 字符串, 同时 key 和 value 都应转为 string 类型. 其中定义有以下常量:

  • pkg_type 数据包类型
pkg_type name description
00 PKG_HEART 客户端发送心跳
01 PKG_HEART_ACK 服务端响应心跳
02 PKG_DEVICE_CHECK 客户端设备检查
03 PKG_DEVICE_CHECK_ACK 服务端响应设备检查
04 PKG_DEVICE_BIND 客户端设备绑定
05 PKG_DEVICE_BIND_ACK 服务端响应设备绑定
06 PKG_DEVICE_PUSH 服务端推送消息
07 PKG_DEVICE_PUSH_ACK 客户端推送消息回执
  • respcd 返回码
respcd name description
0 ACK_OK 请求成功
-1 ACK_DEVICE_FORBIDDEN 设备
-2 ACK_PARAM_ERR 请求参数错误
-3 ACK_TOKEN_ERR 设备未绑定
-4 ACK_SIGN_ERR 签名错误

1. 签名方式.

对待签名数据数据中的每一项, 按照 key 以字典序排序, 并拼接成 key=value&key2=value2 格式的字符串, 在最后拼接上 secret_key, 计算出 MD5 作为签名.

例如, 对于 secret_key=abcdef, 数据:

{
    "pkg_type": "04",
    "pkg_id": "1",
    "apptype": "412",
    "userid": "123",
    "deviceid": "123456",
    "platform": "android",
    "platform_ver": "6.0.1",
    "sdk": "pushcore"
}

应拼接成:

apptype=412&deviceid=123456&pkg_id=1&pkg_type=04&platform=android&platform_ver=6.0.1&sdk=pushcore&userid=123abcdef

计算出的 MD5 为 d1d593ac98e0decaef776116e82cbcf4.

2. 设备绑定.

  • Request
{
    "pkg_type": "04",
    "pkg_id": "客户端数据包 id",
    "apptype": "绑定的 apptype",
    "sign": "签名",

    "userid": "用户 id",
    "deviceid": "设备唯一标示",
    "platform": "平台类型, 如 android",
    "platform_ver": "平台版本",
    "sdk": "使用的推送 SDK, 如 pushcore"
}
  • Response
{
    "respcd": "0",  // 0 表示成功, 其他情况表示失败.
    "pkg_type": "当 respcd == 0 时为 05, 其他情况时返回请求的 pkg_type",
    "pkg_id": "服务端数据包 id",
    "pkg_ack_id": "该响应对应客户端数据包 id",
    "sign": "签名"
}

3. 发送心跳

  • Request
{
    "pkg_type": "00",
    "pkg_id": "客户端数据包 id",
    "apptype": "绑定的 apptype",
    "sign": "签名"
}
  • Response
{
    "respcd": "0",  // 0 表示成功, 其他情况表示失败.
    "pkg_type": "当 respcd == 0 时为 01, 其他情况时返回请求的 pkg_type",
    "pkg_id": "服务端数据包 id",
    "pkg_ack_id": "该响应对应客户端数据包 id",
    "sign": "签名"
}

4. 接收推送消息

{
    "pkg_type": "06",
    "pkg_id": "服务端数据包 id",
    "sign": "签名",

    "device_id": "设备 id",
    "msg_id": "消息 id",
    "msg": "消息正文"
}

5. 接收消息回执

客户端收到消息后需发送回执给服务端, 否则服务端会在一段时间后尝试重发消息.

{
    "pkg_type": "07",
    "pkg_id": "客户端数据包 id",
    "apptype": "绑定的 apptype",
    "sign": "签名",

    "msgid": "消息 id",
    "arrival_time": "收到消息时间, 10 位时间戳"
}
